# nPos N6 — แผนเฟสย่อย (โคลน UX เว็บ + ตรวจอัตโนมัติ)

หลัก: เครื่องอยู่ไกล · พนักงานไม่ใช่ช่าง · **อย่าให้พนักงานต้องสแกน/วินิจฉัยเอง**  
หลังร้าน + agent อ่านไทม์ไลน์/สถานะเครื่องแล้วแก้ APK รอบถัดไป

อ้างอิง UX ที่ต้องโคลน: `PosSellView` · `PosShiftView` · `PosAppShell` · `pos-sales` / `pos-sync`

---

## เปิดแอปแล้วเจออะไร (เป้าหมายหลัง N6)

```
เปิดแอป
  │
  ├─ (พื้นหลังอัตโนมัติ — พนักงานไม่เห็นศัพท์เทคนิค)
  │     heartbeat · เช็คอัปเดตเงียบๆ · สแกนฮาร์ดแวร์เบาๆ · ส่ง ops/diagnose
  │
  ├─ ยังไม่เปิดรอบ  →  หน้าเข้างาน / เปิดรอบ (โคลนเว็บ)
  │
  └─ เปิดรอบแล้ว   →  หน้าขาย: หมวด → เมนู → ตะกร้า → ชำระ
                        แถบล่าง/เมนู: รอบการขาย · ใบเสร็จ · ตั้งค่า
```

| สิ่งที่พนักงานเห็น | สิ่งที่ซ่อน / อัตโนมัติ |
|--------------------|------------------------|
| ขาย · ชำระ · เปิด/ปิดรอบ | สแกน USB/BT/จอ |
| สถานะสั้นๆ “ออนไลน์ / รอซิงก์ N บิล / มีอัปเดต” | รายละเอียด error, vendor id, stack |
| ตั้งค่า (เฉพาะที่จำเป็นหน้าร้าน) | ตรวจเครื่องเต็มรูปแบบ (หลังร้านอ่านจาก log) |

ปุ่ม **ตั้งค่า** ยังมี — แต่เป็น “ตั้งค่าหน้าร้าน” ไม่ใช่ห้องแล็บเทคนิค

---

## ตั้งค่าอยู่ตรงไหน

### A) ในแอป (พนักงานแตะได้น้อย)

| ส่วน | เนื้อหา | ใครใช้ |
|------|---------|--------|
| **ตั้งค่ากิจการ** | ชื่อบิล · PromptPay · พิมพ์อัตโนมัติหลังขาย | เจ้าของ/หัวหน้าสาขา |
| **เครื่องนี้** | รหัสเครื่อง · เวอร์ชัน · สถานะออน · “มีอัปเดต — กดอัปเดต” | พนักงานกดอัปเดตได้ |
| **ปริ้นเตอร์** | เลือกเครื่องที่ระบบเจอแล้ว · ทดสอบพิมพ์/ลิ้นชัก (ปุ่มเดียว) | ตอนติดตั้งครั้งแรก หรือเมื่อเปลี่ยนเครื่อง |
| **เกี่ยวกับ** | ลิงก์หน้าติดตั้ง | หายาก |

**ย้ายออกจากสายตาพนักงาน (หรือพับลึก + รหัสเจ้าของภายหลัง):**  
ตรวจจอทีละจอ · สแกนฮาร์ดแวร์ดิบ · ยอดทดสอบจอลูกค้าแบบมือ · log ดิบ  
→ ระบบทำอัตโนมัติแล้วส่งหลังร้านแทน

### B) หลังร้าน (เจ้าของ / agent)

| หมวดพับ | ใช้ทำอะไร |
|---------|-----------|
| เครื่อง nPos (ออนไลน์) | เห็นเครื่องไกล · เวอร์ชัน · printerReady |
| ไทม์ไลน์ nPos (ops log) | error อัตโนมัติ → แก้ APK |
| ตรวจเครื่อง (nPos) | snapshot จอ/USB/BT ล่าสุดจากสแกนอัตโนมัติ |

## ตรวจ / ส่งกลับอัตโนมัติ (บังคับทุกเฟสย่อย N6)

| เมื่อไร | ส่งอะไร | ไปที่ |
|--------|---------|-------|
| เปิดแอป / กลับโฟกัส | heartbeat + `printerReady` | `posDevices` |
| ทุก ~6 ชม. หรือหลังบูต | diagnose เบา (จอจำนวน · USB/BT count · IP) | `nposDiagnose` |
| error จอ/พิมพ์/ลิ้นชัก/ซิงก์/อัปเดต | ops event สั้นๆ | `nposOpsLog` |
| หลังขาย / ปิดรอบพลาดพิมพ์ | ops `printer` + ถ้าต้องการ retry เงียบ 1 ครั้ง | ops log |
| เมนูโหลดไม่ได้ / sync ค้าง | ops `sync` / `menu` | ops log |

พนักงานเห็นแค่ข้อความมนุษย์ เช่น “พิมพ์ไม่สำเร็จ — ร้านจะดูให้อัตโนมัติ”  
**ไม่โชว์** exception ยาวบนจอขาย

## เฟสย่อย N6

### N6.0 — เปลือกขาย + สแกนอัตโนมัติ (ก่อนเมนูจริง)
- เปลี่ยนหน้าแรกจาก Hello World → **เข้างาน/เปิดรอบ หรือขายว่าง**
- พื้นหลัง: auto heartbeat · auto diagnose เบา · auto ops
- ตั้งค่าจัดกลุ่มใหม่ตามตารางด้านบน (เทคนิคลึกพับ/ลด)
- เกณฑ์ผ่าน: เปิดแอปแล้วพนักงานเจอหน้าขาย/เข้างาน · หลังร้านมี log โดยไม่มีคนกด “ส่งผล”

### N6.1 — เมนูท้องถิ่น
- ดึง/แคชเมนู+หมวดจาก Firestore (หรือไฟล์ซิงก์) เก็บในเครื่อง
- UI โคลน: แถบหมวด + กริดเมนู
- ออฟไลน์แล้วยังเห็นเมนูล่าสุดที่แคชไว้
- เกณฑ์ผ่าน: เปิดดูเมนูได้โดยไม่พึ่งเว็บ · โหลดพลาดมี ops log

### N6.2 — ตะกร้า + ตัวเลือก + ส่วนลด
- โคลนโปรเซสเว็บ: แตะเมนู → ตัวเลือก → ตะกร้า ±/ลบ → ส่วนลด
- จอลูกค้า (ถ้ามีจอ 2) อัปเดตยอดตะกร้าอัตโนมัติ (ผูก N3)
- เกณฑ์ผ่าน: สร้างตะกร้าทดสอบครบ · error จอลูกค้าเข้า ops โดยไม่ต้องกดเทส

### N6.3 — ชำระเงิน (เงินสด / PromptPay ยืนยัน)
- โคลน pay modal: ทบทวนออเดอร์ · กดชำระ · เงินสด keypad หรือ PromptPay QR + ยืนยันขาย
- บันทึกบิล **local-first** (outbox) แล้วค่อย sync
- เกณฑ์ผ่าน: ขายเทสครบ 1 บิลออฟไลน์แล้วโผล่หลังร้านเมื่อมีเน็ต

### N6.4 — พิมพ์ใบเสร็จ + ลิ้นชักอัตโนมัติหลังขาย
- หลังขายสำเร็จ: พิมพ์ ESC/POS อัตโนมัติ (ถ้าเปิดในตั้งค่า) · kick ลิ้นชักเมื่อเงินสด
- ล้มเหลว → retry 1 ครั้งเงียบ + ops log · พนักงานเห็นปุ่ม “พิมพ์อีกครั้ง” เท่านั้น
- เกณฑ์ผ่าน: ขายเงินสดแล้วกระดาษออก / ลิ้นชักเปิด โดยไม่เข้าเมนูเทส

### N6.5 — รอบการขาย + ใบเสร็จย้อนหลัง + นำร่องร้าน
- โคลนเปิด/ปิดรอบ · Z/snapshot พิมพ์ · ประวัติใบเสร็จในเครื่อง
- ใช้จริงคู่ขนานเว็บ 1–2 เครื่อง · เว็บเป็น fallback
- เกณฑ์ผ่าน: ปิดรอบได้ · หลังร้านเห็นยอดตรง · ops ไม่มี error ซ้ำร้ายแรง

*(N7 แยกต่างหาก — ตัดเส้นทางเว็บขายหน้าร้าน)*

## สิ่งที่ยังไม่อยู่ใน N6

- สมาชิก / พักบิล / โต๊ะ (ของที่เว็บยังไม่เปิดจริง)
- สต็อก / OT / รายงานเจ้าของบนแท็บเล็ต
- LAN printer `:9100` (เพิ่มทีหลังถ้าจำเป็น — USB/BT ก่อน)

## ลำดับลงมือแนะนำ

1. **N6.0** ก่อน — โครงนำทาง + สแกนอัตโนมัติ (คุ้มค่าสุดสำหรับเครื่องไกล)
2. N6.1 → N6.2 → N6.3 (แกนขาย)
3. N6.4 (ฮาร์ดแวร์ผูกขาย)
4. N6.5 (รอบ + นำร่อง)

แต่ละย่อย = APK เวอร์ชันใหม่ + เกณฑ์ผ่านชัด + อ่านได้จากไทม์ไลน์หลังร้านโดยไม่ถามพนักงาน
