# nPos — เฟสย้ายจาก POS Web → Native

เป้าหมาย: ค่อยๆ ย้ายความสามารถจาก `telltea-pos` (WebView / browser POS) มาแอป **nPos-telltea** โดยมีระบบเทสฟังก์ชันในเครื่อง + รายงานกลับหลังร้าน + **ไทม์ไลน์ ops** ให้แก้เวอร์ชันถัดไปได้โดยไม่ต้องถามรายละเอียดเทคนิคจากร้าน

## ทำไม POS web เดิมยังอยู่

| เหตุผล | รายละเอียด |
|--------|------------|
| ร้านยังขายอยู่ทุกวัน | เว็บ POS คือเส้นทางขายจริงตอนนี้ — ตัดทันที = ร้านหยุดขาย |
| Native ยังไม่ครบวงจรขาย | ยังขาดเมนู/ตะกร้า/ชำระ/ซิงก์บิล (N6) |
| ลดความเสี่ยง | เทสทีละเฟสบนเครื่องจริง (จอ → ปริ้น → ลิ้นชัก → ขาย) แล้วค่อยตัดเว็บ |
| หลังร้านยังใช้เว็บ | รายงาน / จัดการ Pos / ตั้งค่า ยังอยู่บน TellTea web |

**แผนโคนราก (ตัดเว็บขายหน้าร้าน):**

1. **ตอนนี้–N5** — native เป็น “เครื่องทดสอบฮาร์ดแวร์ + อัปเดต” คู่ขนานกับเว็บขาย
2. **N6** — native ขาย offline/local-first ได้ครบรอบทดสอบ (เมนู→ตะกร้า→บิลเทส→ซิงก์)
3. **N6.5** — ร้านนำร่องใช้ nPos เป็นหลัก 1–2 เครื่อง · เว็บเป็น fallback
4. **N7** — ปิดเส้นทางขายหน้าร้านบนเว็บ/WebView · เหลือเว็บแค่หลังร้าน + หน้าติดตั้ง APK

จนกว่า N6 ผ่านเกณฑ์ — **อย่าลบ** `telltea-pos` / เส้นทาง `/pos/sell`

## โคลน UX จาก POS web — ไม่ redesign ใหม่

**ยังไม่ทิ้งงานออกแบบเว็บ** — N6 ต้อง **พอร์ตโปรเซส + หน้าจอขาย** จากของเดิม ไม่เริ่ม layout ใหม่จากศูนย์

| KEEP (โคลนมาใช้) | REPLACE (ชั้น native เท่านั้น) | DROP (ไม่เอาเข้า nPos ขาย) |
|------------------|-------------------------------|---------------------------|
| โครงหน้าขาย: หมวด → เมนู → ตะกร้า → ชำระ | พิมพ์ ESC/POS USB/BT (แทน `window.print`) | Chrome print dialog / PWA install |
| เปิดรอบ / ปิดรอบ / ประวัติใบเสร็จ | ลิ้นชัก kick ผ่านปริ้นเตอร์ | Capacitor WebView เป็นผลิตภัณฑ์หลัก |
| ตัวเลือกเมนู · ส่วนลด · เงินสด / PromptPay | จอลูกค้า Presentation ผูกตะกร้า | หลังร้านรายงาน/สต็อก/OT (อยู่ TellTea web) |
| local-first outbox → sync บิล | อัปเดต APK `latest.json` | ฟีเจอร์ค้างที่เว็บยังไม่เปิดจริง (สมาชิก/พักบิล ฯลฯ) |

อ้างอิงหน้าเว็บที่ต้องตาม: `PosSellView` · `PosShiftView` · `PosAppShell` · `pos-sales` / `pos-sync` · `pos-printer` (template เก็บไว้ เปลี่ยนแค่ driver)

## หลักการ

- **เว็บ POS เดิมยังไม่ลบ** จนกว่า native ผ่านเทสทีละฟังก์ชัน
- แต่ละเฟสต้องมี: สิ่งที่ตรวจในแอป · สิ่งที่ส่งกลับหลังร้าน · เกณฑ์ผ่าน
- อัปเดต APK ผ่าน `latest.json`
- เครื่องมือเทสอยู่ใน **ตั้งค่า** (หน้าแรก = brand + รหัสเครื่อง + heartbeat)
- **Ops log** — error / ผลฮาร์ดแวร์สั้นๆ ส่งกลับไทม์ไลน์หลังร้านอัตโนมัติ
- **เครื่องไกล** — สแกน/วินิจฉัยทำพื้นหลังให้มากที่สุด พนักงานไม่ต้องมีความรู้เทคนิค

## Ops log (ข้ามเฟส · ตั้งแต่ 1.6.0)

ส่งอะไรกลับ (สั้น · ไม่ใช่ไฟล์ใหญ่):

| cat | ตัวอย่าง |
|-----|---------|
| `display` | แสดงยอด/เลขบนจอสำเร็จหรือ “การแสดงบนจอไม่สำเร็จ…” |
| `printer` | พิมพ์ทดสอบสำเร็จ/ล้ม · ปลายทาง USB/BT |
| `drawer` | สั่งเปิดลิ้นชักสำเร็จ/ล้ม |
| `hardware` | สแกนปริ้นเตอร์ / ส่งผล diagnose |
| `heartbeat` / `update` | ลงทะเบียนหรืออัปเดตไม่สำเร็จ |

Collection: `nposOpsLog/{installId}` — ring buffer ~60 events  
Diagnose snapshot ยังอยู่ที่ `nposDiagnose/{installId}`  
หลังร้าน: **POS → จัดการ Pos → ไทม์ไลน์ nPos (ops log)**

เป้าหมาย: agent อ่านไทม์ไลน์แล้วแก้ APK รอบถัดไปได้ โดยร้านไม่ต้องอธิบายศัพท์เทคนิค

## หลายเครื่องทั้งที่เทสเครื่องเดียว (แก้แล้ว)

`ANDROID_ID` + `stableKey` · CF มาร์ก sibling `disabled` · หลังร้านซ่อนซ้ำ/ออฟ

## หลังร้าน

- **เครื่อง nPos (ออนไลน์)**
- **ไทม์ไลน์ nPos (ops log)**
- **ตรวจเครื่อง (nPos)**

## นอกขอบเขตรอบนี้

- ยังไม่ผูกยอดจอลูกค้ากับตะกร้าขายจริง
- ยังไม่มี LAN `:9100` discovery / driver แยกจาก USB·BT
- ยังไม่ตัดเว็บขาย (รอ N6–N7)
