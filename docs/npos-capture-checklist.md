# nPos — D1 สเปกจอ + C1 แคปจอหลังร้าน

เป้า: ส่งพนักงานได้เมื่อรู้สเปกจอครบ + สั่ง/ตั้งเวลาแคปจากหลังร้านได้

อัปเดต: **1.12.1** · `APP_BUILD` 224

## เช็คลิสต์

### D1 — สเปกจอในรายงาน
- [x] `DisplayProbe` ส่ง width/height/dpi/orientation/rotation
- [x] AutoHealth / diagnose CF / หลังร้านโชว์สเปกจอ 1–2
- [x] สรุปสถานะจอลูกค้า: `ok` / `missing` / `orientation` hint

### C1 — แคปจอผ่านรายงาน
- [x] หลังร้านกด **สั่งแคปจอ** → CF `nposOwnerDeviceCommand` (Admin) เขียน `captureRequestAt`
- [x] heartbeat ตอบกลับคำสั่งแคปให้นาทีฟ
- [x] แคปจอหลัก (หน้าต่างแอป) + จอสอง (Presentation ทดสอบสั้น)
- [x] อัปโหลดผ่าน CF `reportNposScreenCapture` → Storage + `nposScreenShots`
- [x] หลังร้านเห็นภาพล่าสุดในแผงตรวจเครื่อง

### C2 — ตั้งช่วงเวลา
- [x] หลังร้านตั้ง `captureIntervalMinutes` (0=ปิด, 5, 10, 30) ผ่าน CF เดียวกัน
- [x] นาทีฟแคปตามช่วงเมื่อแอปเปิดอยู่ (ผ่าน heartbeat)

### สิทธิ์เครื่อง (1.12.1)
- [x] เปิดแอปครั้งแรก / ปุ่ม **อนุญาตสิทธิ์ทั้งหมด** — Bluetooth · แจ้งเตือน · ติดตั้งอัปเดต · แบตเตอรี่
- [x] heartbeat ส่ง `permissionsOk` / `permissionsStatus` ให้หลังร้านเห็น
- [x] แคปจอแอปเองใช้ PixelCopy — **ไม่ต้อง** MediaProjection / สิทธิ์จอทั้งเครื่อง

### หมายเหตุ error «สิทธิ์ไม่พอ» ตอนสั่งแคป
ข้อความนั้นมาจาก **Firestore/เจ้าของหลังร้าน** ไม่ใช่สิทธิ์ Android บนแท็บเล็ต  
แก้แล้วด้วย CF Admin (`nposOwnerDeviceCommand`) + กว้าง rules `isOwnerEmail`
