# nPos / POS — จัดลำดับขายดี + โหมดเรียงเมนู

อัปเดต: **1.14.11** · `APP_BUILD` 241 · `POS_BUILD` 63  
แนวคิด: **เก็บสถิติขายดีไปเรื่อยๆ** · **จัดลำดับที่หลังร้าน** · **หน้า POS แสดงตามโหมด** (fix หรือ กลุ่มขายดี)

## ความเข้าใจที่ล็อก

| หัวข้อ | ความหมาย |
|--------|----------|
| **เก็บข้อมูล** | นับ qty ขายต่อ `menuItemId` (+ หมวด) ไปเรื่อยๆ — ไม่ใช่แค่รายงานวันเดียว |
| **หน้าต่าง** | ช่วงแรก **7 วัน** · พอมีข้อมูลนิ่งขยายเป็น **14 วัน** ได้ (ค่าคอนฟิกหลังร้าน) |
| **ที่คำนวณ** | **หลังร้าน (BO / Cloud)** รวมแล้วเขียน **ตารางจัดลำดับ** `meta/posMenuRank` |
| **Local-first** | เครื่องนับตอนขายสำเร็จทันที → ซิงก์ขึ้นหลังร้าน → ดึงตาราง rank มากับเมนู/แคช |
| **จังหวะเลื่อน** | **ไม่เลื่อนทุกบิลกลางจอขาย** — ใช้ตอนเปิดกะ / รีเฟรชเมนู / ซิงก์ snapshot |
| **โหมดเรียง (หลังร้านตั้ง 2 แบบ)** | **`fix`** = ลำดับคงที่/มือ · **`bestsellers`** = กลุ่ม+เมนูขายดีจริงอัตโนมัติ |
| **หน้า POS** | โหมด `bestsellers` แสดง **ขายดีของจริงเท่านั้น** (จากตาราง rank) — ไม่ปั้นจากธง `recommended` |
| **อย่าทับ `sortOrder` ถาวร** | แยกชั้นแสดง (`salesRank` / display order) · มือลากในโหมด fix ยังใช้ได้ |

## โหมดเรียง 2 รูป (หลังร้านเลือก 1)

### 1) `fix` — ลำดับคงที่ / มือ
- ใช้ `sortOrder` + fixed category names (ถ้ายังเปิด)
- ลากจัดที่ BO / หมวดบน POS ได้ตามเดิม
- **ไม่** ใช้สถิติขายดีจัดหน้า

### 2) `bestsellers` — กลุ่มขายดี
1. รู้ **หมวดขายดี** และ **เมนูขายดี** จากตาราง rank (หน้าต่าง 7→14 วัน)
2. **หมวดขายดีขึ้นหน้า**
3. **ในแต่ละหมวด** เมนูขายดีขึ้นบน
4. เสมอกัน → `sortOrder` มือ → ชื่อ
5. ยังไม่มีสถิติ → ต่อท้าย
