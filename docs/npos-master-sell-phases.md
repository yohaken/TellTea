# nPos — ขายหน้าร้านแบบเรียบ (ขาเร็ว)

อัปเดต: สcope ร้าน TellTea — **ไม่ฟุ่มเฟือย**  
อ้างอิงจอลูกค้า: [npos-customer-display-checklist.md](./npos-customer-display-checklist.md)

## นโยบายร้าน (ล็อกสcope)

| ทำ | ไม่ทำ |
|----|--------|
| ช่องทาง **หน้าร้านอย่างเดียว** | ไม่แยก ทานที่ร้าน / รับกลับ |
| **ใบเสร็จใบเดียว** ให้ลูกค้า + พนักงานชงตามนั้น | ไม่มีสลิปครัว / KDS |
| จิ้มเมนู → จ่าย → พิมพ์ → บิลถัดไป | ไม่มีคิวครัวแยกจอ |
| จอลูกค้าช่วยโชว์รายการ/QR | ไม่บังคับดูใบเสร็จทุกแก้ว — ทำเร็วได้ |

ร้านลักษณะขาเร็ว: บางทีแทบไม่ต้องดูใบเสร็จก็ชงได้ — ระบบต้องสั้น ไม่กดหลายชั้น

## Layout พนักงาน (ที่มี / ปรับเบาๆ)

| พื้นที่ | เป้า | ตอนนี้ |
|--------|------|--------|
| ซ้าย | หมวด + กริดเมนู + เลือก option | มีแล้ว (option เป็นไดอะล็อก — ใช้ได้ ไม่ต้องย้ายแผง) |
| ขวา | ตะกร้า + ยอด + จ่าย | weight 35 (1.14.3) |

**ปรับแล้วแบบเบา:** ซ้าย 65 / ขวา 35 — ไม่ทำระบบสเกลซับซ้อน

## ไหลขายจริง (สั้น)

```
จิ้มเมนู (+ option) → ตะกร้า / จอลูกค้าอัปเดต
        → แก้จำนวนถ้าต้อง
        → จ่าย PromptPay (QR จอลูกค้า) หรือ เงินสด (รับ/ทอน + ลิ้นชัก)
        → ยืนยัน → พิมพ์ใบเสร็จ → รีเซ็ตพร้อมบิลใหม่
```

| ขั้น | สถานะ |
|-----|--------|
| 1 จิ้มออร์เดอร์ | ✅ หมวด/เมนู/option/ตะกร้า/จอลูกค้า |
| 2 แก้ในตะกร้า | ✅ +/- จำนวน · ❌ โน้ตบิล (ไม่จำเป็น — ไม่ใส่ในคิวหลัก) |
| 3 จ่าย | ✅ QR จอลูกค้า · ✅ เงินสดรับ/ทอน (สองปุ่มก็พอ ไม่ต้องรวม Pay) |
| 4 ยืนยัน | ✅ มือกดยืนยัน · ✅ ลิ้นชักตอนสด · ❌ auto ตัดยอด QR (ไม่เร่ง — ยืนยันมือพอ) |
| 5 ใบเสร็จ + พร้อมบิลใหม่ | ✅ พิมพ์ใบเสร็จ + รีเซ็ตตะกร้า/จอ Idle · **ไม่มีครัวแยก** |
