## 1) เป้า Layout — POS Master Screen

| พื้นที่ | สัดส่วน | เนื้อหา |
|--------|---------|---------|
| **ซ้าย** | 60–70% | หมวด · กริดเมนู · ตัวเลือก/ท็อปปิ้ง |
| **ขวา** | 30–40% | ตะกร้า · ยอดรวม · ปุ่มชำระ |

**สถานะตอนนี้ (1.14.x)**  
- ✅ โครงซ้ายเมนู + ขวาตะกร้ามีแล้ว (`activity_sell.xml` · ตะกร้าคงที่ ~344dp)  
- ⚠️ ยังไม่ใช่เปอร์เซ็นต์ 60–70 / 30–40 แบบยืดหยุ่นทุกจอ  
- ⚠️ ตัวเลือก/ท็อปปิ้ง = ไดอะล็อก ไม่ได้อยู่ในคอลัมน์ซ้าย  
- ✅ จอลูกค้าซิงก์ตะกร้า / QR / เงินสด / สำเร็จ แล้ว

## 2) เป้า Process 5 ขั้นตอน × สถานะโค้ด

| ขั้น | เป้า | nPos ตอนนี้ | ช่องว่างหลัก |
|-----|------|-------------|--------------|
| **1 Take Order** | ประเภทออร์เดอร์ · หมวด→เมนู→option · ตะกร้า+จอลูกค้า | หมวด/เมนู/option/ตะกร้า/จอลูกค้า ✅ | ทานที่ร้าน / รับกลับ ❌ |
| **2 Review** | ทวนรายการ · โน้ต · แก้จำนวน | แก้จำนวน ✅ | โน้ตพิเศษ ❌ · ทวนรายการตอนจ่ายบางส่วน · โชว์ option ในตะกร้าบางส่วน |
| **3 Payment** | Pay → QR บนจอลูกค้า / เงินสดรับ-ทอน | QR จอลูกค้า ✅ · keypad เงินสด ✅ | รวมปุ่ม Pay เดียวแล้วเลือกช่องทาง (ตอนนี้แยกสองปุ่ม) |
| **4 Confirm** | QR auto cut-off · เงินสด+ลิ้นชัก | ลิ้นชักตอนขายสด ✅ · ยืนยันมือทั้ง QR/สด | **ตรวจยอดเข้า PromptPay อัตโนมัติ ❌** |
| **5 Print & Dispatch** | ครัว/KDS · ใบเสร็จลูกค้า · รีเซ็ต | ใบเสร็จลูกค้า+รีเซ็ต+จอ Idle ✅ | **สลิปครัว / KDS ❌** |

## 3) เฟสผสานถัดไป (ลำดับแนะนำ)

เรียงตาม dependency + มูลค่าใช้งานร้าน — ไม่ประมาณวัน

### M1 — Master layout ยืดหยุ่น (พนักงาน)
**เป้า:** ซ้าย 60–70% / ขวา 30–40% + auto-resize คล้ายจอลูกค้า  
- [ ] เปลี่ยนตะกร้าจาก `344dp` คงที่ → `layout_weight` (เช่น 35) + ซ้าย weight 65  
- [ ] สเกลปุ่ม/กริดตามความกว้างจอพนักงาน (`DisplayMetrics` จอหลัก)  
- [ ] (ทางเลือก) แผงตัวเลือกติดใต้กริดหรือ sheet ในคอลัมน์ซ้าย แทนไดอะล็อกเต็มจอถ้าจอกว้าง  
- [ ] ทดสอบ tablet แนวนอน + emu  

### M2 — Take Order ครบช่องทาง
**เป้า:** ขั้น 1 ใกล้ร้านจริง  
- [ ] สลับ **ทานที่ร้าน / รับกลับ** บนหัวตะกร้าหรือใต้หมวด (ค่า default ทานที่ร้าน)  
- [ ] เก็บ `orderChannel` ในบิล local + ส่ง `nposCompleteSale`  
- [ ] โชว์ชื่อ option/ท็อปปิ้งใต้บรรทัดตะกร้า  
- [ ] จอลูกค้า receipt แสดง channel สั้นๆ (เช่น «รับกลับ»)  

### M3 — Review & โน้ต
**เป้า:** ขั้น 2  
- [ ] ช่อง **โน้ตบิล** (เช่น ไม่ใส่น้ำแข็ง / แพ็คแยก)  
- [ ] ส่งโน้ตลงใบเสร็จ + payload ขาย  
- [ ] (ทางเลือก) แผงทวนรายการก่อนจ่าย — รายการเต็มใน sheet ยืนยัน  

### M4 — Payment UX
**เป้า:** ขั้น 3 ลื่นขึ้น  
- [ ] ปุ่ม **ชำระเงิน** หลักเดียว → เลือก PromptPay / เงินสด  
- [ ] คงพฤติกรรมจอลูกค้า: QR ทับ media · เงินสดโชว์รับ/ทอน  
- [ ] ส่วนลด / ถือบิล อยู่ในขั้น Review ชัดเจน  

### M5 — Confirm อัตโนมัติ (QR) + ลิ้นชัก
**เป้า:** ขั้น 4  
- [ ] คง: เงินสดกดยืนยัน → `drawerKick` (มีแล้ว)  
- [ ] **ออกแบบ** ช่องทางตรวจยอด PromptPay (webhook ธนาคาร / slip verify / poll สถานะ) — ยังไม่มีในระบบ  
- [ ] ระหว่างรอ QR: สถานะ «รอเงินเข้า» บนจอพนักงาน + จอลูกค้า  
- [ ] Timeout / ปุ่มยืนยันมือสำรอง (ร้านตัดเองได้เสมอ)  
- [ ] ล็อกไม่ให้ซ้ำบิลตอน auto cut-off  

### M6 — Print & Dispatch
**เป้า:** ขั้น 5  
- [ ] สลิปครัว ESC/POS (พอร์ตจากเว็บ `buildKitchenJobs` / `autoPrintKitchen`)  
- [ ] ตั้งค่าเครื่องพิมพ์: ใบเสร็จ vs ครัว (endpoint คนละเครื่องได้)  
- [ ] (เฟสถัดไป) KDS หน้าจอครัว — ไม่บล็อกเปิดร้านถ้ามีสลิปครัวก่อน  
- [ ] หลังพิมพ์สำเร็จ → ตะกร้าว่าง · จอลูกค้า Idle · พร้อมออร์เดอร์ถัดไป (ส่วนใหญ่มีแล้ว)  

### M7 — นำร่องร้าน + ตัดเว็บขาย
- [ ] P4 เทสเครื่องจริงสองจอ + ปริ้นเตอร์  
- [ ] P5 เก็บ feedback (layout / QR / ครัว)  
- [ ] P6 void เซิร์ฟเวอร์  
- [ ] N7 ตัดเว็บขายหลัง M1–M6 นิ่ง

## 4) เช็คลิสต์มือ (ร้าน / เครื่องจริง)

ใช้หลังจบแต่ละเฟส M1–M6

| # | สถานการณ์ | ผ่านเมื่อ |
|---|-----------|-----------|
| H1 | เปิดขายสองจอ | พนักงาน = master · ลูกค้า = Idle สองพาเนล |
| H2 | เลือกหมวด→เมนู→option | เข้าตะกร้าขวา + จอลูกค้าอัปเดตทันที |
| H3 | ทานที่ร้าน / รับกลับ | สลับได้ · โผล่ในบิล/ครัว |
| H4 | แก้จำนวน + โน้ต | ตะกร้าและจอลูกค้าตรงกัน |
| H5 | จ่าย PromptPay | QR ใหญ่บนจอลูกค้า · ยืนยัน (หรือ auto) แล้วสำเร็จ |
| H6 | จ่ายเงินสด | คีย์รับ · ทอนถูก · ลิ้นชักเด้ง |
| H7 | หลังขาย | ใบเสร็จลูกค้า · สลิปครัว (เมื่อมี M6) · จอ Idle พร้อมบิลใหม่ |

## 5) Dependency สั้นๆ

```
M1 layout ──┬──► M2 channel / options ในตะกร้า
            └──► M4 Pay UX
M2 + M3 ────────► M5 QR auto (ต้องมี payload บิลครบ)
M4 + ลิ้นชักที่มี ─► M5 สำรองมือ
เว็บ kitchen print ─► M6 native kitchen
M1–M6 นิ่ง ────────► M7 ตัดเว็บขาย
```

**ไม่บล็อกเปิดร้าน:** M5 auto cut-off และ KDS เต็มรูป — ใช้ยืนยันมือ + สลิปครัวก่อนได้
