# nPos / POS — ไหลขายพนักงาน (สกัดจากโปรเซสทั่วไป → ร้านเรา)

อัปเดต: วางแผน · ยังไม่ชิปโค้ด  
แหล่ง: โปรเซสสั่ง–ชำระ 5 ขั้นที่คุย + นโยบายหน้าร้านขาเร็ว

## โปรเซสของเรา (ย่อ — ใช้แค่นี้)

```
เลือกหมวด → จิ้มเมนู → option
  → แก้จำนวนในตะกร้า (± / ลบ)
  → (ถ้ามี) ส่วนลด % หรือ บาท
  → จ่าย สด หรือ PromptPay (ยืนยันมือ)
  → พิมพ์ตามตั้งค่า → รีเซ็ตบิลถัดไป
```

## ตัดออกแล้ว (ไม่ทำ · ไม่ใส่คิว)

| หัวข้อจากโปรเซสทั่วไป | เหตุผลร้านเรา |
|------------------------|----------------|
| **ทานที่ร้าน / รับกลับ / Delivery** | หน้าร้านอย่างเดียว · **ตัดป้าย/ปุ่ม/ช่องทางทั้งหมด** |
| โน้ตรายการในตะกร้า | นอกสcope · option พอใช้ |
| คูปอง / สมาชิก / สะสมแต้ม | ยังไม่มีระบบจริง (stub ไม่เปิด) |
| บัตรเครดิต / โอน / อื่นๆ | จ่ายได้แค่สด + PromptPay |
| QR auto ตัดยอดธนาคาร | ยืนยันมือตามนโยบาย |
| ตัดสต็อกจาก POS | โดเมนห้าม |
| สลิปครัว / KDS | ใบเสร็จลูกค้าใบเดียว |

## เฟสงานที่สกัดได้

### F0 ตัด “ทานที่ร้าน” ออกจาก UI
- [ ] เว็บขาย: เอาป้าย/ข้อความ `ทานที่ร้าน` · `orderChannel: dine_in` ที่โชว์พนักงานออก (ไม่โชว์เป็นตัวเลือกช่องทาง)
- [ ] ใบเสร็จ/รายการบิลย้อนหลัง: ไม่โชว์แถว “ทานที่ร้าน” เป็นช่องทาง
- [ ] native: ตรวจว่าไม่มีป้ายช่องทางค้าง · ถ้ามีให้เอาออก
- [ ] เอกสาร/เช็คลิสต์: ย้ำว่าช่องทางออร์เดอร์ **ตัดแล้ว**

### F1 ตะกร้าขาเร็ว (ขั้น Take Order ที่ขาด)
- [ ] ปุ่ม **ล้างตะกร้า** (ยืนยันสั้นๆ) — เว็บ + native
- [ ] native: **แก้ option ในตะกร้า** ได้ใกล้เว็บ (แตะบรรทัด → เปิด picker ใหม่)
- [ ] ลบรายการชัดเจน (ไอคอนถังหรือลดถึง 0 แล้วหาย — ให้พฤติกรรมสองฝั่งใกล้กัน)

### F2 จ่ายให้ครบสองช่องทาง
- [ ] เว็บขาย: ปุ่ม **PromptPay** จากตะกร้าใช้งานได้เท่า native (ตอนนี้ logic มีแต่ CTA ค้าง)
- [ ] สด: คีย์แพด + พอดี/ทอน — คงของเดิม · ตรวจว่าใช้ลื่นทั้งสองฝั่ง
- [ ] PromptPay: ยืนยันมือ · ยกเลิกกลับไปเลือกสดได้ · **ไม่** ทำ auto refresh ธนาคาร

### F3 ส่วนลด (ขั้น Review แบบเบา)
- [ ] คงส่วนลด **%** และ **บาท** — ไม่แยกหน้า Review ยาวถ้าไม่จำเป็น
- [ ] ปิด/ซ่อนปุ่มสมาชิก·คูปองที่เป็น stub ไม่ให้เข้าใจว่าใช้ได้
- [ ] native: ส่วนลด % ให้ชัดเท่าเว็บถ้ายังอ่อน

### F4 จบรายการ (Complete)
- [ ] หลังขาย: รีเซ็ตตะกร้า + จอลูกค้า Idle (มีแล้ว — ยืนยันไม่พัง)
- [ ] พิมพ์ตาม `autoPrintReceipt` · พิมพ์ซ้ำจากใบเสร็จย้อนหลังพอ — ไม่บังคับปุ่มพิมพ์/ไม่พิมพ์ทุกบิล
- [ ] **ไม่** เพิ่มสลิปครัว / ตัดสต็อก

### F5 คนเทสเคาน์เตอร์
- [ ] เดินโปรเซสสั้น 10–20 บิล: จิ้ม → option → ส่วนลด → สด/QR → บิลถัดไป
- [ ] ยืนยันว่าไม่มีคำว่าทานที่ร้าน/รับกลับบนจอขาย

## ลำดับแนะนำลงมือ

1. **F0** ตัดทานที่ร้าน (สั้น · ลดความสับสน)  
2. **F1** ล้างตะกร้า + แก้ option native  
3. **F2** PromptPay เว็บ  
4. **F3** เก็บส่วนลด / ซ่อน stub  
5. **F4–F5** ยืนยันจบรายการ + คนเทส
