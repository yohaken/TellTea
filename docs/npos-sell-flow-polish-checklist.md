# nPos / POS — ไหลขายพนักงาน (สกัดจากโปรเซสทั่วไป → ร้านเรา)

อัปเดต: **1.14.12** · `APP_BUILD` 243 · `POS_BUILD` 64  
แหล่ง: โปรเซสสั่ง–ชำระ 5 ขั้นที่คุย + นโยบายหน้าร้านขาเร็ว

## โปรเซสของเรา (ย่อ — ใช้แค่นี้)

```
เลือกหมวด → จิ้มเมนู → option
  → แก้จำนวนในตะกร้า (± / ลบ)
  → (ถ้ามี) ส่วนลด % หรือ บาท
  → จ่าย สด หรือ PromptPay (ยืนยันมือ)
  → พิมพ์ตามตั้งค่า → รีเซ็ตบิลถัดไป
```

## ตัดออกแล้ว (ไม่ทำ · ไม่ใส่คิว)

| หัวข้อจากโปรเซสทั่วไป | เหตุผลร้านเรา |
|------------------------|----------------|
| **ทานที่ร้าน / รับกลับ / Delivery** | หน้าร้านอย่างเดียว · **ตัดป้าย/ปุ่ม/ช่องทางทั้งหมด** |
| โน้ตรายการในตะกร้า | นอกสcope · option พอใช้ |
| คูปอง / สมาชิก / สะสมแต้ม | ซ่อน stub จนกว่ามีระบบจริง |
| บัตรเครดิต / โอน / อื่นๆ | จ่ายได้แค่สด + PromptPay |
| QR auto ตัดยอดธนาคาร | ยืนยันมือตามนโยบาย |
| ตัดสต็อกจาก POS | โดเมนห้าม |
| สลิปครัว / KDS | ใบเสร็จลูกค้าใบเดียว |

## เฟสงาน

### F0 ตัด “ทานที่ร้าน” ออกจาก UI
- [x] เว็บขาย: เอาป้าย `ทานที่ร้าน` ออกจากตะกร้า
- [x] ใบเสร็จ/รายการบิลย้อนหลัง: ไม่โชว์แถวช่องทาง · ตัดแท็บ ทานที่ร้าน/กลับบ้าน/เดลิเวอรี่
- [x] พิมพ์ใบเสร็จ: ไม่พิมพ์ badge ประเภทช่องทาง
- [x] native: ไม่มีป้ายช่องทาง · ซ่อนเมนูสมาชิกในแถบซ้าย

### F1 ตะกร้าขาเร็ว
- [x] ปุ่ม **ล้างตะกร้า** (ยืนยันสั้นๆ) — เว็บ + native
- [x] native: **แก้ option ในตะกร้า** (แตะบรรทัด → เปิด picker ใหม่)
- [x] ลบรายการชัดเจน (ปุ่ม ×) ทั้งสองฝั่ง

### F2 จ่ายให้ครบสองช่องทาง
- [x] เว็บขาย: ปุ่ม **เงินสด** + **PromptPay** จากตะกร้า
- [x] สด: คีย์แพด + พอดี/ทอน คงของเดิม
- [x] PromptPay: ยืนยันมือ · ยกเลิกได้ · ไม่ทำ auto ธนาคาร

### F3 ส่วนลด + ซ่อน stub
- [x] ส่วนลด % / บาท คงไว้
- [x] ซ่อนปุ่มสมาชิก · ส่งค้างไว้ stub · ตัดจาก nav เว็บ/native
- [x] native: ส่วนลด % presets → บาท (มีอยู่แล้ว)

### F4 จบรายการ
- [x] หลังขาย: รีเซ็ตตะกร้า + จอลูกค้า Idle
- [x] พิมพ์ตาม `autoPrintReceipt` · ไม่บังคับปุ่มพิมพ์ทุกบิล
- [x] ไม่เพิ่มสลิปครัว / ตัดสต็อก

### F5 คนเทสเคาน์เตอร์
- [ ] เดินโปรเซสสั้น 10–20 บิลหลังอัป APK
- [ ] ยืนยันว่าไม่มีคำว่าทานที่ร้าน/รับกลับบนจอขาย
