# nPos — คำถามก่อนส่งพนักงานติดตั้ง (ต้องตอบให้ชัด)

อัปเดต: ตามโค้ดปัจจุบันบนสาขาพัฒนา (production APK สดอาจยังเก่ากว่า — ต้อง merge + publish APK ก่อนส่งจริง)

---

## 1) อัปเดตเองไหมหลังติดตั้งครั้งแรก?

### คำตอบสั้น
**ไม่ต้องกลับไปสแกน QR / เปิดหน้าเว็บทุกครั้ง** — แต่ยัง **ไม่ใช่เงียบอัปเดตเอง 100% โดยไม่แตะอะไร**

### พฤติกรรมจริงตอนนี้
| ขั้น | พนักงานทำอะไร |
|------|----------------|
| ครั้งแรก | เปิดลิงก์/QR → ดาวน์โหลด APK → **อนุญาตติดตั้งจากแหล่งนี้** (ครั้งเดียวต่อเครื่อง) |
| รอบถัดไปที่มีเวอร์ชันใหม่ | เปิดแอป → **ตั้งค่า** → ระบบเช็ค `latest.json` เป็นระยะ → เห็นแบนเนอร์ → กด **อัปเดตเลย** |
| บางเครื่อง Android | ระบบอาจเด้งหน้าต่างยืนยันติดตั้งอีกครั้ง (`PENDING_USER_ACTION`) — กดยืนยัน |

### สิ่งที่ยังไม่ทำ
- ไม่ดาวน์โหลด+ติดตั้งเงียบในพื้นหลังโดยไม่ให้พนักงานรู้/กด
- ไม่บังคับอัปเดตตอนเปิดแอปขายอัตโนมัติ (กันสะดุดตอนขาย)

### สรุปสำหรับส่งพนักงาน
“ติดตั้งครั้งแรกจากลิงก์นี้ครั้งเดียว · รอบหลังอัปเดตในแอปที่ตั้งค่า · ไม่ต้องโหลดจากเว็บซ้ำทุกครั้ง — แต่ยังมีปุ่มให้กดเมื่อมีของใหม่”

---

## 2) หลังติดตั้ง ระบบรู้สเปกจอ 1–2 และสเปกเครื่องไหม?

### คำตอบสั้น
**รู้บางส่วนอัตโนมัติแล้ว — ยังไม่ครบพอสำหรับออกแบบจอลูกค้าละเอียด**

### ส่งกลับอัตโนมัติอยู่แล้ว (`AutoHealth` ≈ ทุก 6 ชม. + เปิดแอป)
ไปที่หลังร้าน → **ตรวจเครื่อง (nPos)** / ไทม์ไลน์ ops:

| มีแล้ว | ยังไม่มี (ต้องทำ D1) |
|--------|----------------------|
| จำนวนจอ · ชื่อจอ · จอหลัก/จอรอง · displayId | กว้าง×สูง px ของแต่ละจอ |
| USB ที่เสียบ · BT ที่ pair · เน็ต/IP | densityDpi / refresh |
| เวอร์ชันแอป · รหัสเครื่อง · heartbeat | orientation จริง (แนวตั้ง/นอน) |
| | ผลเทสว่า Presentation โชว์บนจอ 2 ได้ไหม |

### สรุป
พอใช้รู้ว่า “มีกี่จอ / ต่ออะไรอยู่”  
**ยังไม่พอ** ตอบคำถามแบบ “จอสองเป็นแนวนอน 1280×800 เลยขอบไหม” จนกว่าจะขยาย probe

---

## 3) รู้ไหมว่าแสดงผล/แคปจอสองฝั่งถูกต้องหลังสั่งงาน?

### คำตอบสั้น
**ตอนนี้ยังไม่รู้จากภาพ — รู้แค่บางเหตุการณ์ข้อความ**

| ทำได้ตอนนี้ | ยังทำไม่ได้ |
|-------------|-------------|
| สั่งโชว์ยอดบนจอลูกค้า (Presentation) | แคปภาพจอหลัก/จอสองส่งหลังร้าน |
| ops ว่า “อัปเดตจอลูกค้าไม่สำเร็จ” | ตรวจอัตโนมัติว่าตัวหนังสือเลยขอบจอ |
| | ตรวจว่า layout แนวตั้งบนจอแนวนอนเพี้ยนจากพิกเซล |

การ “สั่งให้แสดง แล้วดูผลว่าถูกต้องไหม” แบบที่ต้องการ = ต้องมี **แคปเจอร์ + อัปโหลด + ดูในหลังร้าน** (ยังไม่มีในโค้ด)

---

## 4) ตั้งเวลาแคปจอทุก 5–10 นาทีจากหลังร้านได้ไหม?

### คำตอบสั้น
**ทำได้ในทางเทคนิค — แต่ยังไม่มีในระบบ · ต้องสร้างเฟสใหม่**

### ข้อจำกัด Android ที่ต้องรู้ก่อนสัญญาพนักงาน
1. **แคปจอทั้งเครื่อง** มักใช้ `MediaProjection` → Android บังคับให้ผู้ใช้กดอนุญาตอย่างน้อยครั้งแรก (คล้ายแชร์หน้าจอ)
2. **แคปเฉพาะหน้าต่างที่แอปเราสร้าง** (จอขาย + Presentation จอลูกค้าที่เราถือ) ทำด้วย `PixelCopy` ได้โดยไม่ต้อง MediaProjection ทั้งเครื่อง — เหมาะกับ nPos มากกว่า
3. อัปโหลดภาพถี่ๆ = เน็ต/แบต/ความเป็นส่วนตัว (มีลูกค้าโผล่จอ) → ต้องเปิด/ปิดจากหลังร้าน + ช่วงเวลา + คุณภาพต่ำ/ย่อขนาด
4. ช่วงขายควรเลี่ยงแคปถี่ หรือแคปเฉพาะเมื่อสั่ง / นอกชั่วโมงเร่ง

### โครงที่วางไว้ถ้าจะทำ (ยังไม่ลงโค้ด)
```
หลังร้านตั้งค่า: intervalMinutes = 0|5|10|30  (0 = ปิด)
        │
แอปอ่านค่าตอน heartbeat
        │
ตามกำหนด → PixelCopy จอหลัก + จอลูกค้า (ถ้ามี)
        │
อัปโหลด Storage → meta ใน nposDiagnose / คอลเลกชัน nposScreenShots
        │
หลังร้านเปิดดูภาพล่าสุดต่อเครื่อง
```

## เกณฑ์ “ส่งพนักงานได้” vs “ยังไม่ส่ง”

| เงื่อนไข | สถานะ |
|----------|--------|
| ติดตั้งครั้งแรกจากลิงก์/QR ได้ | ✅ (หลัง merge + publish APK ที่ต้องการ) |
| อัปเดตรอบหลังในแอป ไม่ต้องโหลดเว็บซ้ำทุกครั้ง | ✅ มีแล้ว (ยังมีปุ่มให้กด) |
| อัปเดตเงียบ 100% ไม่ต้องกดเลย | ❌ ยังไม่สัญญา |
| สเปกจอ/เครื่องพื้นฐานเข้ารายงานอัตโนมัติ | ✅ บางส่วน |
| สเปกจอครบ (ขนาด/แนว) สำหรับพัฒนาจอลูกค้า | ❌ รอ D1 |
| แคปจอตรวจ layout / เลยขอบ | ❌ ยังไม่มี |
| ตั้งเวลาแคปจากหลังร้านทุก N นาที | ❌ ยังไม่มี — วางแผนได้ |

**คำแนะนำ:**  
ถ้าเป้าหมายรอบนี้คือ “ให้พนักงานขายบน nPos ได้ + อัปเดตในแอป” → ส่งได้หลัง publish APK ที่ต้องการ  
ถ้าเป้าหมายคือ “เรามองเห็นจอร้านแบบภาพทุก 5–10 นาทีก่อนส่ง” → **ยังไม่ส่ง** จนกว่าจะทำแคปเจอร์เฟสแรกเสร็จอย่างน้อยแบบสั่งมือจากหลังร้าน

---

## ลำดับทำเมื่อปลดล็อกพัฒนาต่อ

1. **D1** — probe ขนาด/orientation ส่ง diagnose (ไม่มีภาพ)  
2. **C1** — สั่งแคปครั้งเดียวจากหลังร้าน (จอหลัก + จอลูกค้าที่เราถือ)  
3. **C2** — ตั้ง interval จากหลังร้าน + ย่อภาพ + เก็บประวัติสั้น  
4. (ทางเลือก) อัปเดตในแอปให้ดาวน์โหลดอัตโนมัติเมื่อว่างรอบ — ยังให้ยืนยันติดตั้งถ้า Android บังคับ
