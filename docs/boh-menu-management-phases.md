# แผนระบบจัดการเมนูหลังร้าน (BOH Menu Management)

> อัปเดต 2026-07-22 · เฟส / โครงอ้างอิงหลัก

## เป้าหมาย

ให้พนักงานหรือเจ้าของร้าน **สร้าง · ลบ · ปรับแต่งเมนู + ตัวเลือก** จากหลังร้าน TellTea โดยไม่ต้องไปสร้างที่เครื่องขาย (POS)  
ข้อมูลเขียน Firestore `menu*` ชุดเดียว → **POS Web / nPos ดึงไปขายเองอัตโนมัติ**

```
  อื่นๆ → เมนู (แหล่งอ้างอิงหลัก · telltea-shop)
              │  เขียน Firestore
              ▼
     menuCategories / menuItems / menuOptionGroups
              │
       ┌──────┴──────┐
       ▼             ▼
  POS Web ขาย    nPos snapshot
  /pos/sell/     (nposMenuSnapshot)
```

---

## สถานะปัจจุบัน (ฐานที่ต่อได้)

| ส่วน | สถานะ |
|------|--------|
| Firestore `menuCategories` · `menuItems` · `menuOptionGroups` | มีแล้ว · ใช้ร่วม POS |
| CRUD + UI จัดการเมนู | อยู่ที่ **POS Web** `/pos/menu/` (`PosMenuAdmin`) |
| หน้าขาย `/pos/sell/` | **ขายอย่างเดียว** (หมด/เรียงหมวด · ไม่มีสร้างเมนู) |
| อื่นๆ (`/more/`) | มี POS รายงาน · ตั้งค่า · **ยังไม่มีการ์ดเมนู** |
| ราคา | ช่องทางเดียว: `price` / `priceDelta` (ยังไม่มีหน้าร้าน vs เดลิเวอรี่) |
| สิทธิ์เขียนเมนู | owner (Google via `getDb`) หรือ POS device (`getPosDb`) — ใช้ `setMenuDbMode` |

### ชี้แจง URL ที่พูดถึง

| URL | บทบาทจริง | หลังระบบเมนูใหม่ |
|-----|-----------|------------------|
| `telltea-pos.web.app/pos/sell/` | **หน้าขาย** (counter) | **คงไว้** — ไม่ลบ |
| `telltea-pos.web.app/pos/menu/` | **จัดการเมนูบน POS** | **deprecate → ตัดออก** เมื่อ BOH พร้อม (น่าจะหมายถึงตัวนี้) |

นโยบายเดิม: เคาน์เตอร์ไม่เปิดทางเข้า BOH/เมนูแอดมิน (`npos-cut-bo-entry`) — จัดการเมนูอยู่ที่ **telltea-shop** เท่านั้น

## โมเดลเป้าหมาย (สรุป)

### เอนทิตี (คง collection เดิม)

1. **หมวดหมู่** (`menuCategories`) — ชื่อ · ลำดับ · เปิด/ปิด  
2. **รายการเมนู** (`menuItems`) — หมวด · ชื่อ · รูป · คำอธิบาย · ผูกกลุ่มตัวเลือก · เปิด/ปิด · โชว์บน POS  
3. **กลุ่มตัวเลือก** (`menuOptionGroups`) — required · single/multi/unlimited · min/max · รายการตัวเลือก embedded  
4. **ผูกเมนู ↔ กลุ่ม** — `menuItems.optionGroupIds[]` (แชร์กลุ่มข้ามเมนูได้)

### ราคา 2 ช่องทาง (ใหม่)

| ระดับ | หน้าร้าน (เดิม) | เดลิเวอรี่ (ใหม่) |
|--------|-----------------|------------------|
| เมนู | `price` | `deliveryPrice` |
| ตัวเลือก | `priceDelta` | `deliveryPriceDelta` |

- ค่าเริ่มต้น: ถ้ายังไม่มีฟิลด์เดลิเวอรี่ → ใช้ราคาหน้าร้าน (backward compatible)  
- บิลขาย snapshot ราคาตอนขาย (ของเดิมมีแล้ว) — เพิ่มช่องทางเมื่อมี order type  
- หน้าขายช่วงแรกยังใช้หน้าร้านอย่างเดียวได้ จนกว่าจะมีช่องทางเดลิเวอรี่บน counter

### พฤติกรรมแบบแอปเมนูอาหารทั่วไป

- สร้าง/แก้ไข/ลบ หมวด · เมนู · กลุ่มตัวเลือก  
- ผูกหลายกลุ่มกับหนึ่งเมนู · เรียงลำดับกลุ่มตอนขาย  
- เปิด/ปิดรายการ (ขายไม่ได้ / หมดชั่วคราว)  
- รูปเมนู · แนะนำ · โชว์บน POS  
- (ภายหลัง) ค้นหา · กรอง · duplicate เมนู

## เฟสงาน

### Phase 0 — ตกลงขอบเขต + เอกสาร (เอกสารนี้)

**ทำ**
- ยืนยันแหล่งอ้างอิงหลัก = หลังร้าน อื่นๆ → เมนู  
- คง `/pos/sell/` · ตัด `/pos/menu/` เมื่อพร้อม  
- ราคา 2 ช่องทางอยู่ในแผน schema (ยังไม่บังคับ ship พร้อม Phase 1)

**เลิกทำในเฟสนี้**
- ไม่แตะ FoodStory sync เป็นตัวหลัก (manual + BOH เป็นหลัก; FS เป็นทางเลือกนำเข้าภายหลัง)

**Done when:** ทีมเห็นด้วยกับเฟสและ schema ทิศทางนี้

---

### Phase 1 — โครงหลังร้านใต้「อื่นๆ」(skeleton + read)

**ทำ**
- การ์ด **เมนู** ใน `/more/` (owner — และ staff ที่มีสิทธิ์ถ้ามี perm ใหม่)  
- Route ใหม่ เช่น `/menu/` หรือ `/menu-catalog/` บน **telltea-shop**  
- หน้า list: หมวด · รายการ · กลุ่มตัวเลือก (อ่านจาก Firestore ชุดเดียวกับ POS)  
- ใส่ `AppShell` prefixes ให้ tab อื่นๆ ไฮไลต์ถูกต้อง  
- Bump `APP_BUILD` เมื่อ ship UI

**ไม่ทำ**
- ยังไม่บังคับ dual price  
- ยังไม่ลบ `/pos/menu/`

**Done when:** เปิดจาก อื่นๆ แล้วเห็นแคตตาล็อกเดียวกับที่ POS ขายอยู่

---

### Phase 2 — CRUD ครบเท่า `/pos/menu/` (parity)

**ทำ**
- สร้าง / แก้ไข / ลบ หมวด · เมนู · กลุ่มตัวเลือก  
- อัปโหลดรูป · ลากเรียงลำดับ · ผูก `optionGroupIds`  
- reuse logic จาก `pos-menu.ts` / `pos-menu-options.ts` / editors ที่มี (ย้ายหรือแชร์คอมโพเนนต์ให้ BOH เป็นหลัก)  
- สิทธิ์: owner เป็นหลัก; ถ้าต้องการพนักงานช่วยจัดการ → เพิ่ม `PermissionKey` เช่น `menuManage`

**Sync**
- ไม่ต้อง pipeline พิเศษ — เขียน `menu*` → `/pos/sell/` preload + `nposMenuSnapshot` รับเอง

**Done when:** เจ้าของสร้างเมนูใหม่จากหลังร้านแล้วขึ้นบน POS ขายได้โดยไม่แตะ `/pos/menu/`

---

### Phase 3 — ตัวเลือกขั้นสูง + UX แบบแอปเมนู

**ทำ**
- ปรับกฎกลุ่ม: required · single/multi/unlimited · min/max ให้ชัดใน UI  
- Duplicate เมนู / คัดลอกกลุ่มตัวเลือก  
- ค้นหา · กรองตามหมวด · ซ่อนรายการ inactive  
- Soft-delete หรือ archive (ถ้ายังไม่อยาก hard-delete ที่อ้างในบิลเก่า)  
- ข้อความช่วยเหลือภาษาไทยสั้นๆ ในหน้าจัดการ

**Done when:** ใช้งานประจำวันได้โดยไม่ต้องเปิด POS Web admin

---

### Phase 4 — ราคา 2 ช่องทาง (หน้าร้าน + เดลิเวอรี่)

**Schema**
```
MenuItem:        price, deliveryPrice?
MenuOptionChoice: priceDelta, deliveryPriceDelta?
```

**ทำ**
- ฟอร์มเมนู/ตัวเลือก: ช่องราคาหน้าร้าน + เดลิเวอรี่  
- migrate อ่าน: ไม่มี `deliveryPrice` → ใช้ `price`  
- อัปเดต `nposMenuSnapshot` + nPos `MenuModels`  
- cart / ใบเสร็จ: เมื่อมี order channel ค่อยเลือกชุดราคา (เว็บ + native)

**ยังไม่บังคับในเฟสนี้ถ้าไม่มีช่องทางขายเดลิเวอรี่บน counter**
- เก็บฟิลด์ไว้ก่อน · UI ตั้งค่าได้ · sell ยังใช้หน้าร้าน

**Done when:** ตั้งราคา 2 ช่องทางจาก BOH แล้ว snapshot/POS อ่านได้ถูกต้อง

---

### Phase 5 — Cutover: ตัดจัดการเมนูออกจาก POS Web

**ทำ**
- Soft deprecate `/pos/menu/`: ลิงก์ไปหลังร้าน หรือ banner “จัดการที่ อื่นๆ → เมนู”  
- ตัด nav / ทางเข้าเมนูแอดมินบน `telltea-pos`  
- หน้า `/pos/sell/` คงไว้: หมดชั่วคราว · เรียงหมวด (ops หน้าเคาน์เตอร์)  
- อัปเดต docs: `pos-versioning.md`, checklist cut-BO  

**ไม่ทำ**
- ไม่ลบหน้าขาย `/pos/sell/`

**Done when:** ไม่มีทางสร้างเมนูบน POS Web แล้ว; BOH เป็นแหล่งเดียว
