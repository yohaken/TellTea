# POS ปิดกะ — ออกแบบระบบและใบพิมพ์

อ้างอิง FoodStory (Snapshot กลางรอบ + รายงานปิดกะ) ปรับให้เหมาะ TellTea POS — local-first, เงินสด + PromptPay, กะยืดหยุ่นข้ามเที่ยงคืน

---

## 1. หลักการ

| หลัก | รายละเอียด |
|------|------------|
| กะ = รอบขาย | เปิด–ปิดด้วยมือ ไม่ตัดอัตโนมัติตามเที่ยงคืน |
| ข้อมูลจริง | บิลใน `posSales` + สำเนาในเครื่อง (`localReceipts`) |
| ปิดกะได้เมื่อ | บิลค้าง sync = 0 และพนักงานยืนยัน |
| แยกจาก OT ปิดกะ | `/ot/` = SOP พนักงานชง · POS = ยอดขายเครื่อง |

---

## 2. ประเภทใบพิมพ์ (3 แบบ)

### A) ใบเสร็จลูกค้า (Receipt) — มีแล้ว

- พิมพ์ทุกบิลขาย / ทำลายบิล
- ไม่เกี่ยวกับปิดกะโดยตรง

### B) Snapshot กลางรอบ (X-Report) — ยังไม่มี

**เมื่อใช้:** ระหว่างกะเปิดอยู่ ต้องการเช็คยอดโดยไม่ปิดรอบ

| หัวข้อ | เนื้อหา |
|--------|---------|
| หัว | ชื่อร้าน, ที่อยู่, โทร, รหัสเครื่อง |
| ประเภท | `Snapshot ระหว่างรอบการขาย` |
| เวลาพิมพ์ | ณ ตอนกดพิมพ์ |
| เปิดรอบ | วันเวลา `openedAt`, กะ (เช้า/บ่าย/ดึก), พนักงาน |
| **ยังไม่มี** | เวลาปิดรอบ, เงินในลิ้นชักจริง, ส่วนต่าง |

**สรุปยอด (Phase 1 — ทำได้จากข้อมูลปัจจุบัน)**

- จำนวนบิล / ยอดรวม
- แยกเงินสด / PromptPay (จำนวนบิล + ยอด)
- บิลรอส่ง (ถ้ามี — แสดงเตือน ห้ามพิมพ์แทนปิดกะ)
- ทำลายบิล (จำนวน + ยอด void)

**สรุปยอด (Phase 2 — ต้องเพิ่มข้อมูล)**

- ยอดตามหมวดเมนู (category)
- เมนูขายดี Top N
- ส่วนลด / โปรโมชั่น
- ช่องทาง (ทานที่ร้าน / เดลิเวอรี่)

**UI:** แท็บกะ → ปุ่ม `พิมพ์สรุปกลางรอบ` (ไม่ปิดกะ)

---

### C) รายงานปิดกะ (Z-Report) — ยังไม่มี

**เมื่อใช้:** หลังกด `ออกงาน (ปิดรอบ)` สำเร็จ — พิมพ์อัตโนมัติ (ถ้าเปิดใน settings)

| หัวข้อ | เนื้อหา |
|--------|---------|
| หัว | เหมือน Snapshot |
| ประเภท | `รายงานยอดการขาย` / `ปิดรอบการขาย` |
| เปิดรอบ | วันเวลา + ผู้เปิด |
| ปิดรอบ | วันเวลา + ผู้ปิด |
| สรุปยอด | เหมือน Snapshot + ครบถ้วนกว่า |

**ส่วนกระทบเงินสด (Phase 2 — สำคัญ)**

```
เงินสดเริ่มต้น     (opening float — กรอกตอนเปิดกะหรือปิดกะ)
+ ยอดขายเงินสด
+ เงินเข้า (cash in)
− เงินออก (cash out)
− คืนเงิน
= จำนวนเงินที่ควรมีในลิ้นชัก

จำนวนจริงในลิ้นชัก  (พนักงานนับแล้วกรอก)
ส่วนต่าง             (จริง − ควรมี)
```

**ส่วนตรวจสอบ**

- ทำลายบิล (void): จำนวนบิล / รายการ / ยอด
- ยกเลิกเมนู (cancel): ถ้ามีในอนาคต

**UI flow ปิดกะ (แนะนำ)**

```
[ออกงาน ปิดรอบ]
    ↓
ตรวจบิลค้าง sync → ถ้ามี → บล็อก
    ↓
Modal สรุปรอบ (ยอด, เงินสด, PP)
    ↓
(Phase 2) กรอกเงินในลิ้นชักจริง + opening float
    ↓
ยืนยันปิดรอบ
    ↓
บันทึก Firestore + local history
    ↓
พิมพ์ Z-Report (ถ้าเปิด auto)
    ↓
กลับหน้า เข้างาน
```

## 3. ข้อมูลที่ต้องเก็บ (Firestore / Local)

### posSessions (มีแล้ว — ขยาย Phase 2)

```ts
{
  deviceId, date, shift, openedAt, closedAt, status,
  saleCount, totalSales,
  cashTotal, promptpayTotal,        // Phase 2: แยกใน session
  openingCash?: number,
  closingCashCounted?: number,
  cashDifference?: number,
  closedByStaffId?: string,
  openedByStaffId?: string,
}
```

### posShiftReports (ใหม่ — Phase 2)

เก็บ snapshot ที่พิมพ์แล้ว (audit)

```ts
{
  sessionId, kind: "snapshot" | "close",
  printedAt, payload: { ...summary },
  deviceId,
}
```

### Local (มีแล้ว)

- `telltea-pos-local-receipts` — บิลในรอบ
- `telltea-pos-local-sessions` — ประวัติรอบปิด (50 รอบล่าสุด)

## 4. โมดูลที่เกี่ยวข้อง

| โมดูล | บทบาท |
|--------|--------|
| `PosShiftView` | UI กะ, KPI, ปิดรอบ |
| `pos-session.ts` | เปิด/ปิด session |
| `pos-local-receipts` | สรุปยอดจากบิลในเครื่อง |
| `pos-printer/receipt-template` | เทมเพลต thermal |
| `pos-sales-report` (หลังร้าน) | รายงานรวมหลายเครื่อง |
| `/ot/` shift-close | SOP พนักงาน — **คนละระบบ** |

## 5. แผนทำเป็นขั้น

### Phase 1 (ทำได้ทันที — ข้อมูลพอ)

- [ ] `buildShiftSnapshotSummary(sessionId)` จาก local receipts
- [ ] เทมเพลต thermal `shift-snapshot-template.ts`
- [ ] ปุ่ม `พิมพ์สรุปกลางรอบ` ใน PosShiftView
- [ ] Z-Report ย่อหลังปิดกะ (ยอดรวม + เงินสด/PP + เวลาเปิด–ปิด)

### Phase 2

- [ ] กรอก opening cash ตอนเข้างาน
- [ ] กรอกเงินในลิ้นชักตอนปิดกะ + คำนวณส่วนต่าง
- [ ] ยอดตามหมวดเมนู / top items
- [ ] Cash in/out ระหว่างกะ
- [ ] บันทึก `posShiftReports` บน Firestore

### Phase 3

- [ ] ช่องทางออเดอร์ (เดลิเวอรี่)
- [ ] ส่วนลด / โปรโมชั่น
- [ ] VAT / service charge (ถ้าร้านใช้)

## 7. สิ่งที่ไม่ทำใน POS

- ไม่ auto-close กะตามเวลา
- ไม่รวม OT โบนัสชงในใบปิดกะ (ลิงก์ไปหลังร้านแทน)
- ไม่บังคับ VAT ถ้าร้านไม่จด
