## เฟสถัดไป (หลัง P0–P5 ที่ ship แล้ว)

### Q1 — ชัดเจนราคาเดลิเวอรี่ + polish ฟอร์มเมนู
**เป้า:** คนใช้เห็นและตั้งราคาเดลิเวอรี่ได้ทันที ไม่สับสนกับราคาหน้าร้าน  

- ฟอร์มเมนู: แถวราคา 2 ช่องชัด (หน้าร้าน | เดลิเวอรี่) · hint “ว่าง = ใช้หน้าร้าน”  
- รายการเมนูในลิสต์: โชว์ทั้งสองราคาเสมอเมื่อมีเดลิเวอรี่  
- ตัวเลือก: หัวคอลัมน์ หน้าร้าน / เดลิเวอรี่ ชัดเจน  
- (เล็ก) รหัสเมนู `code` ถ้ามีใน schema แล้วโชว์ในฟอร์ม  

**ไม่ทำ:** CSV · ช่องทางที่ 3  

### Q2 — จัดการหมวดให้เป็นหน้าจริง
**เป้า:** หมวดใช้งานประจำวันได้แบบตาราง  

- ค้นหาหมวด  
- แสดงจำนวนเมนูต่อหมวด  
- แก้ชื่อหมวดจากลิสต์ / โมดัล  
- เก็บเข้าคลังหมวด (ซ่อน) · ไม่ลบเมนูข้างในโดยอัตโนมัติ  
- คงเรียง ↑↓  

**ไม่ทำ:** กลุ่มหมวดซ้อน · จำกัดเวลาขาย  

### Q3 — ตัวเลือก: ผูกเมนู + สถานะใช้งาน
**เป้า:** ทำงานแบบแอปเมนูทั่วไป — กลุ่มหนึ่งผูกหลายเมนู  

- ลิสต์กลุ่ม: แบดจ์จำเป็น · **ใช้กับ N เมนู**  
- เมนู ⋮ → แก้ไข · **ผูกเมนู** · เก็บเข้าคลัง  
- หน้า/โมดัล **ผูกเมนู**: ค้นหา · ติ๊กหลายเมนู · บันทึกอัปเดต `optionGroupIds`  
- สลับพร้อมขายต่อตัวเลือกในแถว  

**ไม่ทำ:** ตาราง bulk ราคาทุกช่องทางทั้งร้าน (ไป Q4)  

### Q4 — (ทางเลือก) ตั้งราคาตัวเลือกแบบตาราง
**เป้า:** แก้ราคาตัวเลือกข้ามกลุ่มเร็ว  

- ตาราง: ตัวเลือก × หน้าร้าน × เดลิเวอรี่  
- ค้นหา · บันทึกเป็นชุด  
- ปุ่มจัดการช่องทาง = แค่คำอธิบาย 2 ช่อง (หน้าร้าน/ส่ง) ไม่เพิ่มแพลตฟอร์มภายนอก
