# Photo forensics + entry timestamps checklist

เป้าหมาย: รู้ได้มากขึ้นว่าพนักงานเอารูปเก่ามาอัปโบนัสหรือไม่ และให้ทุกรายการโชว์เวลาบันทึกแบบเดียวกับบัญชีพนักงาน

> ความจริง: ถ้าแก้รูปจน EXIF หาย จะรู้แค่เวลาอัปโหลด + lastModified — ไม่ใช่หลักฐานชี้ขาด 100%

---

## Phase 0 — Checklist / ออกแบบ

- [x] สรุปข้อจำกัด EXIF / canvas strip
- [x] ตัดสินใจ: ไม่ใส่คอลัมน์ตาราง — โชว์ในหน้ารายละเอียด + ตอนดูรูปเต็มจอ
- [x] โมเดลเดียวกับ ledger: วันที่รายการ · อัปเดต

## Phase 1 — Entry timestamps (ทุกรายการหลัก)

- [x] `/ledger/` — มีแล้ว + ใช้คอมโพเนนต์ร่วม
- [x] `/production/` — วันที่รายการ + อัปเดต ในฟอร์มแก้ไข/ดู
- [x] `/ot/` (ชง) — เหมือนกัน
- [x] `/owner-books/` — เหมือนกันตอนแก้ไข
- [ ] (ภายหลัง) checklist / tasks ถ้าต้องการ

## Phase 2 — Capture meta ตอนอัปโหลด

- [x] อ่าน EXIF `DateTimeOriginal` **ก่อน** บีบอัด/canvas
- [x] เก็บ `file.lastModified` เป็น proxy อ่อน
- [x] เก็บ `contentHash` (SHA-256 ของไฟล์ต้น) สำหรับจับรูปซ้ำ
- [x] เขียนลง `evidencePhotos`: `capturedAt`, `fileLastModified`, `captureSource`, `contentHash`
- [x] Firestore rules — ฟิลด์เสริมเป็น optional (hasAll เดิมยังใช้ได้)

## Phase 2b — แสดงเวลาของรูป + สัญญาณไม่สอดคล้อง

- [x] ตอนดูรูปเต็มจอ (owner): โชว์ ถ่าย / อัปโหลด
- [x] เทียบกับวันที่รายการ → ข้อความสั้นถ้าวันไม่ตรง (`วันถ่าย≠วันรายการ`)
- [ ] (ภายหลัง) ไฮไลต์ในตารางถ้า mismatch ชัด

## Phase 3 — ลดตัวหนังสือ UI แนบรูป (เบา)

- [x] ตัด hint ยาว / chip «พร้อมอัปโหลด» เหลือเฉพาะตอนออฟไลน์
- [x] ป้ายปุ่มสั้น: ถ่าย / แนบ
- [ ] (ภายหลัง) ย่อข้อความ progress modal ถ้ายังเยอะ

## Phase 4 — ตรวจซ้ำ / รายงาน (ยังไม่ทำรอบนี้)

- [ ] ค้นหารูป `contentHash` ซ้ำข้ามรายการ
- [ ] รายงาน owner: รายการที่วันถ่าย ≠ วันรายการ

---

## วิธีเทสต์รอบนี้

1. เปิดรายการผลิต / ชง / บช.เจ้าของ / บัญชี → เห็น «วันที่รายการ · อัปเดต»
2. แนบรูปใหม่จากแกลเลอรีที่มี EXIF → เปิดดูเต็มจอ (เจ้าของ) เห็นวันถ่าย
3. รูปที่แก้แล้ว EXIF หาย → ยังเห็นเวลาอัปโหลด (`*` = จากไฟล์ ไม่ใช่ EXIF)
4. พนักงาน: ไม่เห็นแถบ forensics เต็มจอ (เฉพาะเจ้าของ)
