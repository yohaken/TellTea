# สารบัญฟังก์ชัน POS — อ้างอิง Wongnai vs TellTea

> **เอกสารอ้างอิงถาวร** — ใช้เทียบก่อนตัดสินใจพัฒนา TellTea POS  
> อ้างอิง: [Wongnai POS basic-setup](https://lmwnmerchantcenter.com/pos-topics/basic-setup) · [POS Classroom](https://lmwnmerchantcenter.com/pos-classroom)  
> อัปเดตล่าสุด: 2026-07-13

---

## หลักการออกแบบ TellTea POS (บันทึกไว้)

- อ่านได้ใน 1–2 วินาที ข้อความสั้น ไม่ลิสต์ยาว
- เรียบง่าย ไม่การ์ดใหญ่ ไม่ซ้อนปุ่มกับข้อความ
- มือถือ/แท็บเล็ตก่อน: เรียงแนวตั้ง ปุ่มเต็มความกว้างด้านล่าง
- รายละเอียดเต็มอยู่ในฟอร์ม ไม่ยัดในหน้าตาราง
- **ยังไม่ทำ POS ขายจริง** จนกว่าจะสั่ง — Phase 0 = เชื่อมต่อเครื่องก่อน

## สรุปสกัด TellTea (~10 ฟังก์ชันหลัก)

### ต้องมี (MVP)

1. ล็อกอิน / ลงทะเบียนเครื่อง
2. เมนู + หมวด
3. รับออเดอร์หน้าร้าน (ตะกร้า)
4. ชำระเงินสด
5. เปิด/ปิดกะขาย
6. พิมพ์ใบเสร็จ
7. ยกเลิกบิล (owner)
8. รายงานยอดขายวัน
9. sync → `ledger`
10. ตรวจสถานะเครื่อง remote

### ควรมี (หลัง MVP)

- QR PromptPay · ปิดเมนูของหมด · แก้ order ก่อนชำระ · ประวัติบิล · เสียงแจ้ง · หักสต็อก

### ไม่ทำ

โต๊ะ · LINE MAN · Mobile Order ลูกค้าสแกน · CRM แต้ม · Multi-channel · เครดิต/แบ่งจ่าย · หลายสาขา · Queue Display

## ลำดับพัฒนา TellTea POS

| Phase | เนื้อหา | สถานะ |
|-------|--------|--------|
| 0 | เชื่อมต่อเครื่อง + auto-update + heartbeat | วางแผนแล้ว ยังไม่ทำ |
| 1 | เมนู + ตะกร้า + เงินสด + sync ledger | รอ |
| 2 | พิมพ์ + สถานะฮาร์ดแวร์ + void | รอ |
| 3 | PromptPay + ปิดเมนูของหมด | รอ |
| 4 | หักสต็อก + รายงาน | รอ |

## สิ่งที่ทำใน TellTea หลักแล้ว (ไม่ต้องซ้ำใน POS)

- **ปิดกะรวม SOP + โบนัสชง** — v110+ บน `/ot/`
  - `src/lib/shift-session.ts` · `src/lib/shift-close.ts`
  - 4 ขั้น: พนักงาน · เช็คเปิดกะ · เช็คปิดกะ · ยอดชง
  - ธงพนักงาน (เหลือกี่ข้อ) + ธงเจ้าของ (คุณภาพข้อมูล)
- บัญชี `ledger` · คลัง `stock` · พนักงาน `staff`

## บริบทเครื่อง POS (บันทึกไว้)

- แท็บเล็ต Android ~15" · RAM 2GB
- พนักงานไม่เก่งเทคนิค — เปิด Shortcut อย่างเดียว
- เจ้าของ remote ไม่อยู่หน้าร้าน
- เปิดค้าง 24 ชม. · พิมพ์ใบเสร็จ · รูปเมนู thumbnail เบาๆ
- โปรแกรมขายเดิมยังใช้อยู่ — POS ใหม่ทดแทนทีหลัง
