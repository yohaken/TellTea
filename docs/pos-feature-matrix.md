# สารบัญฟังก์ชัน POS — Wongnai vs TellTea

> **เอกสารอ้างอิงถาวร** — ใช้เทียบก่อนพัฒนา TellTea POS  
> อ้างอิง: [Wongnai POS basic-setup](https://lmwnmerchantcenter.com/pos-topics/basic-setup) · [POS Classroom](https://lmwnmerchantcenter.com/pos-classroom)  
> อัปเดตล่าสุด: 2026-07-13

---

## หลักการออกแบบ TellTea POS (บันทึกไว้)

- อ่านได้ใน 1–2 วินาที ข้อความสั้น ไม่ลิสต์ยาว
- เรียบง่าย ไม่การ์ดใหญ่ ไม่ซ้อนปุ่มกับข้อความ
- มือถือ/แท็บเล็ตก่อน: เรียงแนวตั้ง ปุ่มเต็มความกว้างด้านล่าง
- รายละเอียดเต็มอยู่ในฟอร์ม ไม่ยัดในหน้าตาราง
- **ยังไม่ทำ POS ขายจริง** จนกว่าจะสั่ง — Phase 0 = เชื่อมต่อเครื่องก่อน

## บริบทร้าน TellTea (อ้างอิง)

- ร้านชา quick service · ไม่ใช่ร้านนั่งทาน
- แท็บเล็ต Android ~15" · RAM 2GB · เปิด 24 ชม.
- พนักงานไม่เก่งเทคนิค — เปิด Shortcut ทิ้งไว้
- เจ้าของ remote — ดูสถานะ + อัปเดตบังคับจากระยะไกล
- โปรแกรมขายเดิมยังใช้อยู่ — POS ใหม่ทดแทนทีหลัง
- TellTea หลัก: https://telltea-shop.web.app
- POS Lite: `/pos/` static เบา — **Phase 0–1 live** (heartbeat + PWA + ขายเงินสด)

### กฎ hosting — ห้ามทำให้หลังร้าน 404

| ห้าม | เหตุผล |
|------|--------|
| สร้าง `public/pos.html` หรือ redirect `/pos` → `pos.html` ใน `firebase.json` | ชนกับ `out/pos/index.html` → loop หรือ 404 (แก้แล้ว v114) |
| แก้ `firebase.json` `public: "out"` หรือ `trailingSlash` โดยไม่ทดสอบ | หลังร้าน `/ledger/`, `/settings/` ฯลฯ พึ่ง `out/<route>/index.html` |
| ลบ route ใน `src/app/` โดยไม่ตั้งใจ | static export จะไม่มี `out/<route>/` |

**ก่อน deploy ทุกครั้ง:** CI รัน `npm run smoke:hosting` ตรวจว่า route หลังร้าน + `/pos/` มี `index.html` และไม่มี `public/pos.html`
