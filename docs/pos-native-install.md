# TellTea POS — ลง Native (APK) แบบตรงๆ

> เป้าหมาย: ลงบน**แท็บเล็ต Android จริงของร้าน** · ข้าม emulator  
> โครงอยู่ใน `android/` แล้ว (app id: `app.telltea.pos`)

## ทำไมไม่ใช้เครื่องจำลอง

- POS ต้องการจอใหญ่ / ทัช / เน็ตจริง / (ทีหลัง) เครื่องพิมพ์
- Emulator ช้า ติดตั้งลำบาก และไม่สะท้อนปัญหาร้าน
- ทางที่จบ = **USB ลงแท็บเล็ต** หรือ **ส่งไฟล์ APK ไปติดตั้ง**

---

## วิธี A — แนะนำ: Android Studio → USB → แท็บเล็ต

### ครั้งเดียวบนแท็บเล็ต
1. ตั้งค่า → เกี่ยวกับเครื่อง → แตะ “หมายเลข Build” 7 ครั้ง จนเปิดโหมดนักพัฒนา
2. เปิด **USB debugging**
3. เสียบ USB เข้าคอม → อนุญาตดีบักบนหน้าจอแท็บเล็ต

### ครั้งเดียวบนคอม (Windows/Mac)
1. ติดตั้ง [Android Studio](https://developer.android.com/studio)
2. เปิดครั้งแรกให้ลง Android SDK / platform-tools
3. Clone repo TellTea แล้ว:

```bash
cd telltea   # รากโปรเจกต์
git pull origin main
npm ci
npm run build
npx cap sync android
npx cap open android
```

4. ใน Android Studio:
   - รอ Gradle sync จบ
   - ด้านบนเลือกอุปกรณ์ = **ชื่อแท็บเล็ต** (ไม่เลือก emulator)
   - กด **Run** (▶)

ได้แอป **TellTea POS** บนหน้าจอหลัก  
ตอนนี้ยังโหลดเว็บสดจาก `https://telltea-pos.web.app/pos/` → แก้เว็บ+deploy แล้วแค่ปิดเปิดแอป / รีโหลด ไม่ต้องบิลด์ใหม่ทุกครั้ง

---

## วิธี B — ไม่มีสาย USB: ส่งไฟล์ APK

หลัง sync สำเร็จ ใน Android Studio:

**Build → Build Bundle(s) / APK(s) → Build APK(s)**

ไฟล์อยู่ประมาณ:

`android/app/build/outputs/apk/debug/app-debug.apk`

1. ส่งเข้าแท็บเล็ต (Drive / LINE / USB ไฟล์)
2. บนแท็บเล็ต: อนุญาตติดตั้งจากแหล่งที่ไม่รู้จัก
3. เปิดไฟล์ APK → ติดตั้ง

อัปเดตรอบถัดไป: ส่ง APK ใหม่ทับ · หรือใช้วิธี A แล้วกด Run ทับของเดิม

---

## ใช้งานหน้าร้านหลังลงแล้ว

| ทำอะไร | ยังไง |
|--------|------|
| เปิดขาย | เปิดไอคอน TellTea POS (ไม่เปิด Chrome) |
| ได้ของเว็บชุดใหม่หลัง deploy | ปิดแอปแล้วเปิดใหม่ หรือปุ่มรีเฟรชในแอป |
| อยากเต็มจอแบบคีออสก์ | เฟสถัดไป (N2) — ยังไม่บังคับรอบแรก |
| เทียบกับของเดิม | PWA ใน Chrome เก็บเป็นสำรองได้ชั่วคราว |

---

## เช็คว่าลงถูก

- ไอคอนชื่อ **TellTea POS**
- เปิดแล้วเข้าหน้า `/pos/` ได้ (กะ / ขาย)
- เวอร์ชันมุมจอขึ้น **POS 56+** หลัง deploy ล่าสุด
- หลังบ้าน (ตั้งค่า → เครื่อง POS) เห็นโหมด **APK** และเลขเปลือกเมื่อเปิดจากแอป native
- ไม่ต้องเปิด Chrome

ถ้าเปิดแล้วขาว/ค้าง:
1. แท็บเล็ตมีเน็ตไหม (รอบนี้แอปยังดึงเว็บจาก Hosting)
2. ลองปิดแอปจาก Recent แล้วเปิดใหม่
3. ลบแอป → ลง APK ใหม่

---

## สิ่งที่ต้องมีบนคอม (สรุป)

- Node 20+
- Android Studio (SDK + platform-tools)
- แท็บเล็ตจริง + สาย USB (วิธี A) **หรือ** วิธีส่ง APK (วิธี B)
- **ไม่ต้อง** พึ่ง Android Emulator
